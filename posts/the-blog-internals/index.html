The blog's internals

June 1, 2022 2-minute read
tutorial
hugo • github

This article describes how the blog is generated and deployed.

Static content generation

To create the blog, I wanted an easy and lightweight solution to setup and deploy. For this reason, I renounced to use a content management system (CMS), such as Joomla or Wordpress, and I have chosen hugo, an open-source static site generator.

For the user interface (UI), the blog is using the Coder theme. The theme is responsive and offers a light or a dark appearance.

Deployment

The blog is hosted on GitHub. I am using two repositories: a public and a private one. Each repository has two branches.

The content of the blog, written in markdown, is pushed to the main branch of the private repository (blog). Then, a GitHub Action is responsible to generate the site and to push the result to a branch called gh-pages.

The GitHub Action to generate the site:

name: github pages

on:
  push:
    branches:
      - main  # Set a branch to deploy
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-20.04
    steps:
      - uses: actions/checkout@v2
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive)
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          # extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        if: github.ref == 'refs/heads/main'
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

The generated website is then pushed to the branch repo-sync of the public repository (smrtnt.github.io) with the help of an additional GitHub Action, repo-sync. The repository is configured to serve this branch for the hosted GitHub Pages.

Statistics

I am using GoatCounter to monitor the web traffic on the blog. GoatCounter is an open-source web analytics platform available for free, for a non-commercial use.

The integration with hugo is simple, only two lines are necessary:

[params.goatCounter]
  code = '< REDACTED >'

Comments

For the comments, I am using a comment system called utterances. Utterances is a lightweight widget built on top of GitHub issues.

The integration in hugo is also simple:

[params.utterances]
  repo = '< REDACTED >'
  issueTerm = 'title'
  label = 'Comment'
  theme = 'github-dark'

Conclusion

The blog is generated with hugo and hosted on GitHub Pages. The content is written in markdown and could be done on several devices, by using git pull and push commands.
